How Super builds and publishes a website

1) Research & structure

Super opens browsers, reviews examples, drafts a sitemap, and validates SEO basics. This mirrors how founders actually work — no magic prompts.

2) Generate real code

She writes HTML/CSS/JS or framework code, creates assets, and previews locally. You can review diffs before anything ships.

3) GitHub & deploy

Super signs into GitHub, commits, opens PRs, and deploys to hosts (with your confirmation). Live previews update as files change.

4) Iterate with cache

Edits, bug fixes, and redeploys reuse a durable computer-use cache, so repeated computer steps don’t start from zero.
